Rapport Final : Checklist de Sécurisation LAMP

Synthèse complète de l'audit et du durcissement effectués sur le serveur.

1. Démarche à suivre (Initialisation)

Préparation de la machine et vérification des versions avant les tests.

Étape 1 : Mettre à jour la machine (Debian/LMDE)

sudo apt update && sudo apt upgrade

Étape 2 : Vérifier les versions installées

apache2 -v
mariadb --version
  • Apache : 2.4.66 (Debian) — 05/12/2025
  • MariaDB : 10.11.14-MariaDB — Distrib 15.1

2. Tableau de Bord de Sécurisation

Élément de sécurité Commande de Test Résultat Attendu Remédiation Appliquée (Solution)
Mise à jour Système sudo apt update && sudo apt upgrade Système à jour Actualisation des dépôts et des paquets Debian/LMDE.
Bannière Apache curl -I http://localhost Server: Apache (sans version) Modifié security.conf (ServerTokens Prod, ServerSignature Off).
Listage Répertoire Accès http://localhost/ 403 Forbidden Supprimé Indexes dans apache2.conf pour /var/www/.
Forçage HTTPS curl -I http://localhost 301 Moved Permanently Ajouté Redirect permanent dans 000-default.conf.
Service HTTPS ss -tunlp | grep 443 Écoute sur le port 443 a2enmod ssl + a2ensite default-ssl. Installation de ssl-cert.
Isolation MariaDB ss -tunlp | grep 3306 127.0.0.1:3306 Modifié bind-address = 127.0.0.1 dans 50-server.cnf.
Accès Root SQL mariadb -u root Access denied (sans sudo) Sécurisation via mysql_secure_installation et plugin unix_socket.
Gestion Privilèges SHOW GRANTS FOR 'admin_web' Error 1141 (Inexistant) Suppression du compte trop privilégié via DROP USER.

3. Rappel des commandes de gestion

Redémarrage

  • Apache : sudo systemctl restart apache2
  • MariaDB : sudo systemctl restart mariadb

Vérification

  • Syntaxe : sudo apache2ctl configtest
  • Logs : sudo tail -n 20 /var/log/apache2/error.log

Administration

  • MariaDB : sudo mariadb
