Rapport Final : Checklist de Sécurisation LAMP

Ce document constitue la synthèse complète de l'audit et du durcissement effectués sur le serveur.

1. Démarche à suivre (Initialisation)

Avant de procéder aux tests de sécurité, la machine doit être préparée et les versions vérifiées.

Étape 1 : Mettre à jour la machine (Debian/LMDE)

sudo apt update
sudo apt upgrade

Étape 2 : Vérifier les versions installées

apache2 -v
mariadb --version
  • Résultat Apache : Apache/2.4.66 (Debian) — Compilé le 05/12/2025
  • Résultat MariaDB : 10.11.14-MariaDB — Distrib 15.1

2. Tableau de Bord de Sécurisation

Élément de sécurité Commande de Test Résultat Attendu Résultat Obtenu Remédiation Appliquée
Mise à jour Système sudo apt update && sudo apt upgrade Système à jour ✅ SUCCÈS Actualisation des dépôts et des paquets Debian/LMDE.
Bannière Apache curl -I http://localhost Server: Apache (sans version) ✅ SUCCÈS Modifié security.conf (ServerTokens Prod, ServerSignature Off).
Listage Répertoire Accès http://localhost/ 403 Forbidden ✅ SUCCÈS Supprimé Indexes dans apache2.conf pour /var/www/.
Forçage HTTPS curl -I http://localhost 301 Moved Permanently ✅ SUCCÈS Ajouté Redirect permanent dans 000-default.conf.
Service HTTPS ss -tunlp | grep 443 Écoute sur le port 443 ✅ SUCCÈS a2enmod ssl + a2ensite default-ssl. Installation de ssl-cert.
Isolation MariaDB ss -tunlp | grep 3306 127.0.0.1:3306 ✅ SUCCÈS Modifié bind-address = 127.0.0.1 dans 50-server.cnf.
Accès Root SQL mariadb -u root Access denied (sans sudo) ✅ SUCCÈS Sécurisation via mysql_secure_installation et plugin unix_socket.
Gestion Privilèges SHOW GRANTS FOR 'admin_web'@'localhost' Error 1141 (Inexistant) ✅ SUCCÈS Suppression du compte trop privilégié via DROP USER.

3. Rappel des commandes de gestion

Redémarrage des services

  • Apache : sudo systemctl restart apache2
  • MariaDB : sudo systemctl restart mariadb

Vérification de configuration

  • Test syntaxe Apache : sudo apache2ctl configtest
  • Logs d'erreurs : sudo tail -n 20 /var/log/apache2/error.log

Accès Administration

  • MariaDB (Admin) : sudo mariadb
