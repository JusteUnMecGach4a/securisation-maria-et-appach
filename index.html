Documentation technique des tests de conformité et des mesures correctives appliquées sur l'infrastructure Debian/LMDE.

sudo apt update && sudo apt list --upgradable

2. Matrice de Sécurisation Avancée

Vecteur de Sécurité Commande de Vérification (CLI) Résultat Attendu & Critères d'Acceptation Remédiation Appliquée
Mise à jour Système apt-get upgrade --dry-run | grep "^Inst" | wc -l Le compteur doit retourner 0. Aucun paquet de sécurité en attente. Patch management global via APT sur dépôts officiels Debian.
Exposition de Bannière curl -I http://localhost | grep -i "Server" Doit afficher uniquement Server: Apache. Aucune version ni OS visible. Restriction via ServerTokens Prod et ServerSignature Off.
Indexation de Répertoire curl -s -o /dev/null -w "%{http_code}" http://localhost/icons/ Retour du code HTTP 403 (Forbidden). L'exploration de l'arborescence est interdite. Désactivation globale de l'option Indexes dans apache2.conf.
Politique HSTS / HTTPS curl -sI http://localhost | grep -E "301|Location" Code 301 Moved Permanently pointant impérativement vers https://.... Directive Redirect permanent / configurée sur le VirtualHost port 80.
Disponibilité SSL/TLS ss -tuln | grep :443 && openssl s_client -connect localhost:443 Port 443 en état LISTEN. Handshake SSL réussi avec certificat valide. Activation du module mod_ssl et configuration du bundle de certificats.
Isolation Réseau SQL netstat -tpln | grep 3306 L'adresse d'écoute doit être 127.0.0.1:3306. Aucune écoute sur l'IP publique. Restriction de la variable bind-address dans la config MariaDB.
Authentification Root SQL mariadb -u root -p'false_pwd' Message : Access denied. L'accès root doit être lié au socket UNIX via sudo. Déploiement du plugin unix_socket et suppression des accès distants.
Moindre Privilège mariadb -e "SELECT User FROM mysql.user WHERE User='admin_web';" Retour vide (Empty set). Le compte utilisateur a été révoqué avec succès. Suppression via DROP USER 'admin_web'@'localhost'.
